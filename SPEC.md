# 랜딩페이지 구현 스펙 (Codex 작업 지시서)

## 결과물
- `index.html` 단일 파일 (CSS/JS 인라인, 외부 CDN·폰트·이미지 요청 금지.
  시스템 폰트 스택 사용: "Pretendard Variable", Pretendard, -apple-system,
  "Malgun Gothic", sans-serif)
- 정적 사이트. GitHub Pages에 그대로 올릴 수 있어야 함.
- 모바일 우선 반응형 (한국 트래픽 대부분 모바일). 가로 스크롤 절대 금지.
- 다크 배경 기반의 프리미엄 테크 느낌. 촌스러운 무지개 그라데이션 금지.

## 제품
한국어 전자책 "나 대신 일하는 AI — Claude Code로 굴리는 1인 자동화 부업 실전 가이드"
- 가격: 14,900원. PDF 60~80쪽.
- 핵심 소구: "ChatGPT에 질문만 하던 사람"을 "AI에게 업무를 통째로 맡기는 사람"으로.
  실제로 운영 중인 광고 자동화 시스템을 해부한 실전서.

## 페이지 구성 (위에서 아래)
1. 히어로: 헤드라인 "AI에게 묻지 말고, 맡겨라", 서브카피, CTA 버튼 "지금 시작하기 — 14,900원"
   (href는 `#buy` 앵커, 실제 결제 URL은 나중에 교체하므로 `data-buy-link` 속성이 있는
   버튼 하나의 href만 바꾸면 되게 구조화)
2. 문제 제기: "ChatGPT 쓰는데 왜 내 삶은 그대로일까" — 3개 공감 카드
3. 해결책: 챗봇 vs 에이전트 비교 표 (2열)
4. 목차 미리보기: 8장 + 부록, 접었다 펴는 아코디언
5. 이 책이 다른 이유: 실운영 시스템 해부 / 복붙 템플릿 / 안전장치 설계 3카드
6. 미리보기 섹션: 본문 일부 발췌 인용 블록 (placeholder 텍스트, 나중에 교체)
7. FAQ 4개 (환불, 난이도, 필요 장비, 업데이트)
8. 최종 CTA (#buy 앵커 위치) + 푸터 (판매자 표기 placeholder)

## 카피 톤
- 과장 금지: "월 천만원", "무조건", "100% 보장" 류 문구 사용 금지.
- 단정하고 구체적으로. 이모지 남발 금지 (섹션당 최대 1개).

## 검수 기준
- 375px 뷰포트에서 레이아웃 깨짐 없음
- 라이트하우스 기준 이미지/폰트 외부요청 0건
- 아코디언/FAQ는 JS 없이 <details>로 구현 권장
